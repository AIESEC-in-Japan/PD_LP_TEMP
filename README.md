# LP自体の説明

## LPとは
LPはLandingPageの略で、顧客（今後ユーザーと呼びます）が最初にたどり着くページです。  
プロダクトの詳細が詰まっている説明ページではありません。  
より詳しいことを知りたい人はここを参照してください。
元KBOGXD大杉さん作成の資料がかなり勉強になります。  
[KBそなえの事例とその関連資料](https://github.com/Yuki-k-lion/sonaeLP/blob/master/info_docks.md#%E3%81%9D%E3%81%AA%E3%81%88lp%E5%BC%95%E3%81%8D%E7%B6%99%E3%81%8E%E3%83%89%E3%82%AD%E3%83%A5%E3%83%A1%E3%83%B3%E3%83%88)
(変なところあれば指摘してください。直します)

### 前提として
webはユーザーは情報にアクセスしやすい一方で、ユーザーから見つけられにくい。  
ユーザーは気軽にどこでもアクセスすることができるが、一方でつまらないとすぐに閉じてしまう。  
必要なのは **Userfirst** の考え方です。  
自分たちが伝えたいことを伝えるのではなく、ユーザーがそのタイミングで知りたいことをわかりやすく伝えることを意識しましょう。

LP作成の際には以下のことを意識すると良いです。
- 顧客が興味を惹かれるポイントはどこか
- 顧客の知りたい情報は何か
- 顧客がこのプロダクトに共感するポイントはどこか
- どれだけのコストがかかるか、かからないか
- なぜ信頼できるのか、信頼できる理由

## Marketingの位置付け
AIDMAモデルを考えると、
1. 無知
2. 認知
3. 興味
4. 比較検討
5. ...

という流れの中で、フェーズ2,3周辺を担います。  
このLPを通してユーザーをフェーズ4に持っていくことを目的とします。

## 事前準備
### 掲載内容の確定
掲載内容を確定する際
以下の設問に完璧に答えられるまで精度を高めてください。  
共通のものは先に書いています。  
1. What:これは何か。
    - LP(LandingPage)
    - NG:なし
2. Why:このページの目的はなにか。
    - NG:プロダクトを知ってもらう。（→それは広報です。販促ではありません。）
3. who:誰がやっているのか。
    - NG:KBメンバー(→どんな想いを持った誰が?)
4. where:どこでみせる？どこで知る？
    - NG:Twitter,Facebook(→もっと具体的に、どこで、どうやって到達するのか)
5. when:いつみせる？いつ見られる？
    - NG:1日中、いつも(→1日中見ている訳ではない。1日中アクセスできる状態と1日中見ている状態はちがう)
6. how:どうやってみせる？どうやってアクセスさせる？
    - NG:Twitterの投稿(→なぜアクセスするのか、そのきっかけは？)
7. how mach:媒体を見るための費用は？
    - 無料
    - NG:なし。
8. How many:何回見る？何日公開？
    - NG:まぁ特になし。
